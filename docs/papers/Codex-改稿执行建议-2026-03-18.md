## 2. 逐段改写策略（按段落/行号）

- `tex/1.intro.tex:24-46`：将三大问题压缩成表格或 bullet，并立刻说明本文如何逐一覆盖，避免“图数据仍未探索”式泛句。
- `tex/1.intro.tex:83-134`：把 RQ1-RQ5 变成“问题+一句话答案+章节指引”的三列表，同时把贡献 bullet 改为“贡献+数量/案例”。
- `tex/2.methodology.tex:5-19`：新增“检索数据源/关键词/时间”、“纳入/排除标准”和“编码流程”，可配流程图。
- `tex/2.methodology.tex:28-74`：为 Figure~\ref{fig:Taxonomy} 补一段“判别标准+代表任务”，并用子小节句式带领读者沿图走读。
- `tex/2.methodology.tex:75-91`：将“100+论文”说明扩展为统计表（按年份/会议），并解释关键词选取方法。
- `tex/3.Preliminaries.tex:12-35`：合并浅层/深层叙述，用表 1 对比“灵活性 vs 表达力”，为后续引出 prompt 铺垫。
- `tex/3.Preliminaries.tex:81-101`：补齐案例（如 All in One、PRODIGY）并删除占位符，使用数学表达式定义“graph transformation = prompt + 原图”框架。
- `tex/4.Pre-training.tex:12-37`：在策略概述后添加“对 prompt 的启示”子段，列出“可复用的预训练目标-提示模板”映射表。
- `tex/5.tex:14-46`：拆成“Token 级/结构级/插入模式”三个表，列出每篇工作的 prompt 参数量、可调部分、任务类型。

- `tex/7.ProG.tex:1-33`：改写为“架构图+核心模块+示例代码+性能基准”，并解释如何复现论文实验。
- `tex/8.Discussion.tex:1-38`：把挑战与未来方向做一一对应（问题、证据、潜在解决方案），并尽量给出量化目标。
- `tex/9.Conclusion.tex:1-5`：扩展为三段：回顾 RQ 结论、概述工具与资源贡献、提出未来研究路径。
