## 一、Towards Graph Prompt Learning: A Survey and Beyond

### 1. 论文定位

这篇论文的目标是给图提示学习建立一个较完整的“学科地图”。它不是只列方法，而是试图回答三个核心问题：

- 图提示学习为什么会出现，它要解决什么矛盾。
- 图提示应该如何拆解和分类。
- 图提示现在能落到哪些应用上，还缺什么理论和评测基础。

它的特点是“面广且系统”。作者把图提示学习放在“预训练-下游适配”的大背景下，把图提示看成弥合预训练图与下游图、预训练任务与下游任务之间差距的机制。

### 2. 论文结构

从正文组织上，这篇论文基本是“背景 -> 方法学基础 -> 具体方法 -> 应用 -> 评测 -> 挑战”的经典综述结构：

1. `Introduction`
   说明图表示学习、预训练、提示学习的发展脉络，强调图域与 NLP/CV 的不同，提出综述动机。
2. `Taxonomy / Terminology / Notations`
   先给名词和符号，随后明确本文的分类体系。
3. `Related Work`
   重点比较图提示学习与图预训练/微调、图少样本学习的关系，也对已有 survey 做差异化说明。
4. `Design Principles`
   这是论文核心，讲 prompt token、任务对齐、prompt tuning、同构图/异构图提示。
5. `Applications`
   讲分子图、文本属性图、推荐系统等。
6. `Benchmark Datasets and Evaluation`
   讨论实验设置、评价标准、数据集使用上的问题。
7. `Open Problems and Future Work`
   讨论理论、评测、统一框架等尚未解决的问题。
8. `Conclusion`

这套结构很像一个“从学科建模到研究议程”的写法。它不是先堆方法，而是先把概念边界讲清楚，再进入分类和应用。

### 3. 研究内容详细讲解

#### 3.1 作者如何理解图提示学习

作者认为图提示学习的核心价值，不是单纯“少调参数”，而是：

- 利用预训练模型中已经获得的图结构知识。
- 通过提示机制，把下游任务重写成更接近预训练目标的形式。
- 以更轻量的方式完成任务迁移，而不必全面微调整个模型。

这个观点和 NLP 中 prompt 的基本哲学一致，但作者明确指出图域更复杂，因为图数据不是线性的，图里同时有节点、边、子图、全图层面的信号，提示本身也不再只是“一段文本”，而可能是节点、结构、子图、任务原型等。

#### 3.2 这篇论文最重要的贡献：设计原则分类

作者把图提示的设计拆成三块：

- `Prompt tokenization`
- `Task alignment`
- `Prompt tuning strategy`

这是全文最值得记住的框架。

`Prompt tokenization` 讲的是“提示本身是什么”。作者分成三类：

- 节点级 token：直接作用在节点特征上，本质是给节点增加可学习偏置或特征补充。
- 结构级 token：把 motif、子图、路径、元路径等结构先验编码成提示。
- 任务级 token：为具体任务设计类别原型、答案向量、任务模板。

这个分类的价值在于，它把“图提示”从一个模糊概念拆成了可设计对象。以后看新方法时，可以先问它主要在改哪个层面。

`Task alignment` 讲的是“如何把预训练目标与下游任务对齐”。作者区分：

- 节点级对齐：常见于节点分类、链接预测，重点是让预训练得到的节点表示更贴近目标节点任务。
- 图级对齐：常见于图分类、子图分类，让全图或子图表示更符合下游标签空间。

这部分的关键观点是：prompt 不是孤立存在的，它的意义来自“对齐”。如果没有对齐设计，prompt 只是额外参数，不一定真的弥合了任务差距。

`Prompt tuning strategy` 则是优化层面的分类。作者区分：

- 基于 pretext 的调优：继续沿用或靠近预训练目标。
- 面向下游任务的调优：直接围绕目标任务损失来学习 prompt。

这部分回答的是“prompt 怎么学”。

#### 3.3 同构图与异构图的分开讨论

这篇论文没有把所有图统一处理，而是单独讨论了：

- 同构图提示
- 异构图提示

这是必要的。因为异构图里不仅节点类型不同，边类型也不同，甚至语义关系本身就是建模对象。所以在异构图中：

- 提示不只是“补特征”，还要编码类型关系。
- 元路径、类型特异提示、跨类型消息传递提示会更重要。

这说明作者不是把图提示当成单一技术，而是认为它会随着图数据形态变化而变形。

#### 3.4 应用部分在讲什么

应用部分覆盖了三个典型方向：

- 分子图
- 文本属性图
- 推荐系统

分子图场景下，作者强调的是结构 motif、官能团、子结构模式的重要性。这里 prompt 的作用往往是把化学先验注入到预训练图模型中。

文本属性图场景下，作者强调图与文本的跨模态融合问题。prompt 的价值在于把文本语义和图结构对齐。

推荐系统场景下，作者强调用户-物品交互图的时序性、语义性和任务差异。prompt 可以作为轻量适配器，帮助模型在不同推荐目标间迁移。

#### 3.5 作者指出的不足

这篇论文最清醒的部分在后半段。作者明确认为当前领域存在几类根本问题：

- 评测不统一，不同论文使用的数据集和设置差异很大。
- 理论解释不足，prompt 为什么有效没有被严格说明。
- 通用性不足，大多数方法对特定图、特定任务高度定制。
- 鲁棒性研究弱，面对图结构扰动、特征攻击时缺少系统分析。
- 与 LLM 的结合还早期，图提示和语言提示的融合机制未成熟。

### 4. 对这篇论文的总体评价

这篇论文适合作为“入门到建立全景图”的第一读物。它的最大优势不是某个单点观点，而是把图提示学习的设计空间组织起来了。缺点也很明显：因为覆盖面大，所以对单个方法族的细节比较克制，更多是在搭框架。

## 二、Graph Prompting for Graph Learning Models: Recent Advances and Future Directions

### 1. 论文定位

这篇论文比上一篇更“工程化”和“方法机制导向”。它的中心不是建立大而全的学科地图，而是围绕一个更明确的主线：

- 图预训练是基础。
- 图提示是适配器。
- 需要系统总结预训练、提示、应用、开放挑战之间的关系。

如果说第一篇偏“领域总览”，这篇更像“围绕 pre-train + prompting 工作流的技术综述”。

### 2. 论文结构

它的组织更紧凑，结构可以概括为：

1. `Introduction`
   说明预训练与下游任务之间存在 objective gap，引出图提示。
2. `Preliminaries`
   给出图定义、图学习模型、问题形式化。
3. `Graph Pre-training for Graph Prompting`
   单独讲生成式、对比式、多任务预训练。
4. `Techniques in Graph Prompting`
   这是核心，按数据级、表示级、任务级提示来分类。
5. `Applications`
   重点讲推荐系统、知识工程等。
6. `Open Challenges`
   讨论 benchmark、理论基础、通用兼容性、鲁棒性、LLM 结合。
7. `Conclusion`

和第一篇相比，它把“图预训练”前置并强化了，这是一个很重要的结构差异。

### 3. 研究内容详细讲解

#### 3.1 这篇论文最核心的叙事：objective gap

作者的出发点非常明确：预训练任务和下游任务经常不一致。

例如：

- 预训练时做链接预测。
- 下游时做节点分类。

这时模型在预训练阶段更关注边关系，而下游阶段更关注节点表征。两者之间天然存在目标错位。作者认为，图提示的价值就是在不大改预训练模型参数的前提下，补这一道“目标鸿沟”。

这个叙事很实用，因为它直接回答了“为什么需要 prompt，而不是继续 fine-tune”。

#### 3.2 预训练部分的作用

第三章不是背景铺垫，而是在回答：图提示依赖什么样的预训练基础。

作者把图预训练分成三类：

- 生成式方法：恢复被破坏的节点、边、属性、结构。
- 对比式方法：通过图增强和正负样本对学习不变表示。
- 多任务方法：同时优化多个预训练任务，让模型学到更综合的图知识。

这部分的重要观点是：prompt 的效果不是凭空产生的，它高度依赖预训练模型中已经压缩了什么知识。如果预训练目标本身就很弱，prompt 的上限也会受限。

#### 3.3 这篇论文的关键分类：按提示插入位置分

作者把图提示方法分成三类：

- `Data-level prompting`
- `Representation-level prompting`
- `Task-level prompting`

这是它和第一篇最大的分类差异。

`Data-level prompting` 关注输入图怎么改。具体又分：

- 基于特征的提示：给节点特征加可学习向量、偏置、实例感知提示。
- 基于插入的提示：往图里插虚拟节点、提示边、提示子图。

这是最接近 NLP prompt 直觉的部分，因为它相当于直接改“输入”。

`Representation-level prompting` 关注隐藏表征或输出表征怎么调。这类方法不一定修改原始图，而是作用在中间层表示、图表示、类别原型或读出层上。

`Task-level prompting` 关注输出任务如何重写。典型做法包括：

- 用链接预测形式重写分类任务。
- 用类别原型或答案向量对齐标签空间。
- 让下游任务更接近预训练时的判别方式。

这个三分法的好处是特别适合分析实现细节。读一篇新 paper 时，可以直接定位它到底在改输入、改表征，还是改任务头。

#### 3.4 这篇论文为什么重视应用

它的应用部分没有铺得太宽，但讲得更像“方法落地”。重点场景包括：

- 推荐系统
- 知识工程
- 以及与 LLM 的结合趋势

推荐系统部分强调图提示在流式推荐、个性化推荐、跨域推荐中的适配作用。这里 prompt 更像轻量任务适配器。

知识工程部分强调知识图谱任务中的任务模板、三元组提示、跨图谱迁移能力。这里 prompt 的意义是帮助模型在不同知识结构和查询模式之间迁移。

这篇论文在应用上给人的感觉是：它更关心 prompt 作为“技术部件”如何嵌入已有图学习系统，而不是单独强调某个领域的理论特性。

#### 3.5 挑战部分的判断

这篇论文最后一章的挑战总结非常有条理，主要是五类：

- `Benchmark construction`
  缺统一、可复现、可公平比较的基准。
- `Theoretical foundation`
  缺解释 prompt 有效性的理论。
- `General compatibility`
  缺跨任务、跨图、跨模型的统一 prompt 框架。
- `Model robustness`
  缺对抗鲁棒性与安全分析。
- `LLM incorporated`
  图提示和 LLM 的结合还处在方法拼接阶段，没有形成稳定范式。

这五类挑战比第一篇更收敛，也更像未来研究议程。

### 4. 对这篇论文的总体评价

这篇论文更适合已经知道图提示概念、现在想建立“方法实现框架”的读者。它比第一篇更聚焦，技术主线更硬，尤其适合理解图提示在 pre-training pipeline 中的位置。相对地，它对图提示的概念哲学、学科边界、全景式分类没有第一篇那么展开。

## 三、GPL-survery

### 1. 论文定位

第三篇是仓库本地生成的综述成稿，对应 `tex_zh` 源文件。从结构和内容看，这篇论文试图做的事情更进一步，不只是总结“图提示有哪些方法”，而是把图提示放进面向 AGI 的大背景中，强调跨模态、跨领域、跨任务的问题。

它的野心比前两篇更大，核心问题不是“图提示怎么用”，而是：

- 为什么 AGI 时代处理图数据会特别困难。
- 图提示是否能成为图数据走向基础模型时代的一种关键接口。
- 如何从统一框架、应用生态、工具库和未来方向上推动整个研究方向。

### 2. 论文结构

从 `tex_zh` 源文件看，结构很完整：

1. `引言`
2. `调研方法`
3. `预备知识`
4. `面向图提示学习的 GNN 预训练`
5. `面向图任务的提示设计`
6. `多模态与多领域中的图提示`
7. `潜在应用`
8. `ProG：统一的图提示学习工具库`
9. `挑战与未来方向`
10. `结论`

这个结构和前两篇相比有两个显著变化：

- 多了“调研方法”章节，说明作者有意把这篇综述做成更规范的系统化综述。
- 多了“工具库 ProG”和“多模态/多领域”章节，说明作者不只讲算法，也讲研究基础设施和更广的 AGI 视角。

### 3. 研究内容详细讲解

#### 3.1 引言部分的立论方式

这篇论文的引言不是传统的“图学习发展史”，而是从 AGI 的挑战出发。作者明确提出三类核心难题：

- 跨模态
- 跨领域
- 跨任务

作者的判断是：NLP 和 CV 已经在这些方向上取得显著突破，但图数据上的 AGI 仍然很初级。原因在于图数据天然复杂：

- 结构不规则。
- 节点和边特征异质。
- 不同图域之间的语义空间和结构模式差异大。

因此，图提示被作者看作一种可能帮助图模型获得“统一适配能力”的机制。

#### 3.2 这篇论文的研究问题意识非常强

引言中明确提出了五个研究问题：

- 如何在统一框架下理解已有工作。
- 为什么使用提示，图提示的本质是什么。
- 如何设计图提示。
- 如何将图提示部署到真实世界应用中。
- 图提示面临哪些挑战与未来方向。

这使得整篇文章不是单纯按章节堆材料，而是围绕研究问题展开。阅读时可以把每章都看成是在回答一个 RQ。

#### 3.3 调研方法章节的作用

这一章在前两篇中都不突出，但这里被单独拿出来，说明作者希望增强综述的规范性。它主要做三件事：

- 说明调研目标。
- 给出 taxonomy。
- 概览文献分布。

也就是说，这篇论文不只是“我读了很多 paper”，而是想展示“我是按什么准则组织这些 paper 的”。

#### 3.4 预备知识与本体解释

第三章的价值不只是补基础，而是为后面关于“图提示本质”的讨论做铺垫。章节包括：

- 符号记号
- 图表示学习
- 预训练与微调
- 提示学习简史
- 为何使用提示：基于灵活性与表达能力的新视角

这里最重要的是最后一节。作者试图从“灵活性”和“表达能力”重新理解 prompt，不把它仅仅看成弥补任务差距的技巧，而是把它视为图模型能力边界的一部分。

这是与前两篇最明显的思想差异之一。前两篇更强调 prompt 的适配作用，这篇则更强调 prompt 与模型表达能力之间的关系。

#### 3.5 预训练章节的写法

第四章把图预训练按任务粒度拆开：

- 节点级策略
- 边级策略
- 图级策略
- 多任务预训练

这说明作者希望把“图预训练为图提示服务”讲得更细，而不是停留在 generative / contrastive / multi-task 的粗分类上。这样的写法更有助于和具体下游提示机制对齐。

#### 3.6 提示设计章节是全文方法核心

第五章包括三块：

- 提示 token、结构与插入模式
- 通过回答函数实现任务对齐
- 提示调参

这相当于把图提示设计拆成三个工程问题：

- prompt 长什么样
- prompt 如何把任务改写对齐
- prompt 怎么训练

和第一篇相比，这篇更强调“回答函数”。这说明作者不只关心 prompt 作为输入修饰，而是关心最后任务如何被重新表述、如何从模型输出中读答案。

#### 3.7 多模态与多领域章节体现其 AGI 视角

第六章单独讲：

- 含图的多模态提示
- 基于提示的图领域自适应

这非常关键。它把图提示从“图学习内部技术”提升成“图与其他模态、其他领域沟通的接口”。作者显然认为图提示未来的重要性，不只在节点分类或图分类，而在于：

- 图和文本/图像/分子语言的对齐。
- 不同图域之间的迁移。
- 图基础模型与通用基础模型的衔接。

这是第三篇最有前瞻性的部分。

#### 3.8 ProG 工具库章节说明作者在建设研究生态

第七章单独介绍 `ProG` 工具库。这说明作者的目标不仅是总结已有工作，还希望推动：

- 统一实验接口
- 统一研究基座
- 结果复现与方法对比

这比纯综述更进一步，因为它试图解决该领域一个长期问题：方法很多，但实现分散、复现成本高、比较不公平。

#### 3.9 挑战与未来方向

第八章分成“当前挑战”和“未来方向”。结合前文，可以推断作者关注的挑战主要集中在：

- 图提示的理论解释
- 图提示在多模态、多领域迁移中的稳定性
- 统一 benchmark 和工具链
- 图基础模型与 AGI 结合
- 从研究 demo 到真实应用的落地能力

### 4. 对这篇论文的总体评价

第三篇的最大特点是“视野最大、研究议程最完整”。它不像第二篇那样紧贴 pre-train + prompting pipeline，也不像第一篇主要做学科地图整理，而是试图把图提示学习提升为 AGI 时代图数据接口问题。它更适合用来思考未来课题，而不仅是回顾现状。

## 四、三篇论文的横向对比

### 1. 关注重点不同

- 第一篇最强的是“系统分类”，适合建立图提示学习的全景认识。
- 第二篇最强的是“技术管线”，适合理解预训练、提示、应用之间的工程关系。
- 第三篇最强的是“研究议程”，适合理解图提示在 AGI 语境下的未来方向。

### 2. 分类方式不同

- 第一篇按 `tokenization / alignment / tuning` 来拆。
- 第二篇按 `data-level / representation-level / task-level` 来拆。
- 第三篇则把 `提示对象、回答函数、调参方式、多模态迁移、工具生态` 放进一个更大的框架。

从研究使用角度看：

- 如果你要做方法归类，第一篇更方便。
- 如果你要写实现或复现路线，第二篇更方便。
- 如果你要找论文选题和未来方向，第三篇更方便。

### 3. 对“prompt 本质”的理解深度不同

- 第一篇把 prompt 主要看成弥合预训练与下游差距的设计机制。
- 第二篇把 prompt 看成适配 objective gap 的技术接口。
- 第三篇进一步把 prompt 和模型灵活性、表达能力、跨域统一性联系起来。

### 4. 对应用与生态的重视程度不同

- 第一篇重应用广度。
- 第二篇重方法落地。
- 第三篇不仅重应用，还重工具库和研究社区基础设施。

---

## 五、阅读结论

如果把三篇论文合在一起看，可以得到一个很清楚的判断：

- 图提示学习已经从“少量 prompt tuning 技巧”发展成一个相对独立的图学习子方向。
- 当前主线问题仍然是如何弥合预训练目标与下游任务之间的差距。
- 真正的难点不只是设计 prompt，而是设计统一的任务对齐机制、评测体系和跨域迁移框架。
- 未来最有潜力的方向，很可能不在单一节点/图分类精度提升，而在图基础模型、多模态图学习、图与 LLM 协同、以及统一工具生态。

如果后续要继续做深入研究，建议把三篇论文分别当作三个入口：

- 第一篇用于搭知识框架。
- 第二篇用于拆技术实现。
- 第三篇用于找研究问题和论文选题。
