
\section{Preliminaries}\label{sec:pre}

This section introduces the background needed for the rest of the survey. We first define the basic notation used throughout the paper, then review the main stages of graph representation learning, including shallow embeddings, supervised GNNs, and pre-training-based transfer. We also briefly revisit the development of prompt learning and explain why prompting deserves separate discussion in graph learning rather than being treated as a direct copy of NLP practice.


\subsection{Notations}\label{subsec:notation}
Let a graph be denoted by $\mathcal{G} = (\mathcal{V}, \mathcal{E})$, where $\mathcal{V} = \{ v_1, v_2, \ldots, v_N \}$ is the node set and $\mathcal{E} \subseteq \mathcal{V} \times \mathcal{V}$ is the edge set. Each node $v_i$ is associated with a feature vector $\mathbf{x}_i \in \mathbb{R}^D$. We use $\mathbf{X} \in \mathbb{R}^{N \times D}$ to denote the node feature matrix and $\mathbf{A} \in \{0,1\}^{N \times N}$ to denote the adjacency matrix, where $\mathbf{A}_{ij} = 1$ if and only if $(v_i, v_j) \in \mathcal{E}$.


\subsection{Graph Representation Learning}
Graph representation learning has developed through several overlapping stages. Early work focused on shallow embedding methods, which map nodes to learnable low-dimensional vectors and are then reused in downstream tasks. Representative examples include DeepWalk~\cite{perozzi2014deepwalk} and node2vec~\cite{grover2016node2vec}. These methods are flexible because the learned embeddings can often be reused across multiple tasks, but they are usually transductive and do not naturally incorporate rich node attributes.


Shallow embedding methods are often grouped by how they define node similarity. One common line is factorization-based~\cite{
% belkin2001laplacian,
% ahmed2013distributed,
% cao2015grarep,
ou2016asymmetric, zhang2019prone}; another is based on random walks~\cite{grover2016node2vec, perozzi2014deepwalk, wang2019hyperbolic, wang2022common}. These methods remain useful when the goal is to preserve structural proximity in a compact representation, but their transfer behavior is limited once node attributes, unseen nodes, or task-specific supervision become central.


Graph neural networks (GNNs) address some of these limitations by learning task-relevant representations through parameterized neighborhood aggregation. Most GNNs follow a message-passing framework and use a more expressive encoder than shallow embeddings. Representative architectures include spectral and spatial variants~\cite{defferrard2016convolutional, he2021bernnet, niepert2016learning, hamilton2017inductive, velickovic2018graph, shi2021masked, xu2018how}. In many settings, GNNs outperform shallow embeddings because they can combine graph structure with node features. The trade-off is that they are usually trained for specific tasks and often require supervision or repeated re-training when the task changes.


This contrast motivates the transfer problem that runs through the rest of the survey. Shallow embeddings are flexible but limited in modeling power. Task-specific GNNs are more expressive but less reusable across settings. Pre-training and prompting can both be understood as attempts to improve transfer, albeit through different mechanisms.

\subsection{Pre-training and Fine-tuning}
Pre-training and fine-tuning has become a common transfer paradigm in graph representation learning. The usual workflow is to pre-train a model on large-scale graph data, with or without labels, and then adapt it to downstream tasks by updating model parameters. In graph learning, representative pre-training objectives include graph autoencoding~\cite{
% kipf2016variational,
wang2017mgae}, masked component modeling~\cite{hu2020strategies, rong2020selfsupervised}, and graph contrastive learning~\cite{velickovic2019deep, sun2020infograph}. We review these objectives in detail in Section~\ref{sec:pretrain_method}.


This paradigm improves transfer because the downstream model no longer starts from scratch. Still, fine-tuning remains model-centric: the usual way to adapt to a new task is to update some or all of the model parameters. This can work well, but it also means that transfer often depends on repeated optimization, access to supervision, and a stable match between the new task and the pre-trained model.


\subsection{A Brief History of Prompt Learning}\label{subsec:his_prompt}

Prompt learning emerged as another way to adapt pre-trained models, especially when full fine-tuning became increasingly expensive. Rather than updating the whole model for every downstream problem, prompting changes the input, the task interface, or a small set of learnable prompt parameters so that the downstream problem is presented in a form closer to what the model has already seen during pre-training~\cite{liu2023pretrain}.

In NLP, prompts take several forms. Some are discrete templates such as cloze-style prompts. Others are learned continuous prompts, including prefix-based methods~\cite{lester2021power, li2021prefixtuning}. Prompt construction may rely on manual templates~\cite{
% petroni2019language,
brown2020language, schick2021fewshot, schick2021it}, discrete prompt search~\cite{jiang2020how, haviv2021bertese, shin2020autoprompt, gao2021making}, or prompting directly in the embedding space~\cite{li2021prefixtuning, lester2021power, tsimpoukelli2021multimodal, qin2021learning}. These developments do not replace fine-tuning altogether. Instead, they expand the set of available adaptation mechanisms.

For graph learning, prompting is attractive for a similar reason: it may enable transfer with fewer updated parameters and with a clearer separation between the pre-trained graph encoder and the downstream adaptation module. At the same time, graph prompting is not a direct replica of NLP prompting, because graph inputs are structured objects and prompt design must also account for topology, insertion patterns, and graph-level prediction mechanisms.


\subsection{Why Graph Prompt?}\label{subsec:why_prompt}

% In this part of our study, we aim to understand why prompts are important. We want to uncover the fundamental aspects of graph prompts. What exactly do graph prompts do in graph problems? How do they fit into the complex graphs, and how do they help us achieve the broader goal of creating AI systems that can handle graph data effectively? These questions highlight the significant role that graph prompts play in shaping the future of AI when dealing with graph information.


\textbf{A. Differences between Graph Prompting and Graph Fine-tuning.}
The difference between graph prompting and graph fine-tuning is easiest to state in terms of where adaptation happens. Fine-tuning is primarily a model-level adaptation strategy: a pre-trained graph model is adjusted so that its parameters better fit a downstream task. Prompting, by contrast, is primarily an input-level or data-level adaptation strategy: the downstream graph is modified through prompt tokens, prompt structures, or prompt-induced interfaces before it is processed by a pre-trained model. In practice, graph prompting may still involve a small number of trainable parameters, but the central idea is not to re-optimize the full encoder for each new task.

% From the whole picture of AIG, there are many techniques for model-level consideration (like model structure design, training, tunning, etc.) Intuitively, these model-level tricks are more like an iceberg on the surface, while the underwater part, that is data, still heavily relies on some primitive hand-crafted work. Compared with the model-level technique, we have very limited learnable approaches to help us handle data-level issues. Graph prompting is just a very notable choice.  

\textbf{B. Prompts as Learnable Graph Modifications.}
One useful way to understand graph prompting is to view it as a learnable approximation to graph transformation. Let $\varphi^*$ denote a pre-trained graph model, $\mathcal{G}$ the original graph, $\mathcal{G}_p^*$ a prompt graph or prompt module, and $\psi(\mathcal{G}, \mathcal{G}_p^*)$ the prompted graph produced by inserting the prompt into the input. Let $\mathbf{g}(\mathbf{A}, \mathbf{X})$ denote a target transformation of the original graph structure and node features. Then, at a conceptual level, graph prompting can be written as
\begin{equation}\label{equ:error_bound_new}%\vspace{-0.05in}
    \varphi^*\left(\psi(\mathcal{G}, \mathcal{G}_p^*)\right)=\varphi^*(\mathbf{g}(\mathbf{A}, \mathbf{X})) + O^{*}_{p\varphi},
\end{equation}
where $O^{*}_{p\varphi}$ is the residual discrepancy between the prompted input and the ideal transformation under the pre-trained model $\varphi^*$. This expression should not be read as a formal guarantee. It is a conceptual lens: prompts can be interpreted as a learnable way to adjust graph structure, node features, or task interfaces so that the downstream input better matches the behavior already captured by the pre-trained encoder.


\textbf{C. Uses Beyond Task Adaptation.}
This perspective also explains why graph prompting may be useful beyond ordinary task transfer. If a prompt can act as a learnable graph modification, then it may be used to adjust the input before inference, for example by compensating for structural mismatch, introducing auxiliary prompt nodes, or aligning a graph with another modality. In principle, this opens the door to applications beyond standard downstream adaptation, including domain alignment, multi-modal interfaces, and robustness-oriented input adjustment. Whether these uses help in practice remains task-dependent, but the key point is that prompting changes the way adaptation is expressed: not only through model updates, but also through learnable modifications to the input graph itself.

In short, graph prompting is worth separate study because it changes the unit of adaptation. Instead of relying solely on repeated model re-training, it offers a way to reuse a pre-trained graph encoder while expressing downstream adaptation through prompt parameters, prompt structure, and prompted graph construction. That shift is exactly what makes graph prompting both flexible and technically distinct from conventional fine-tuning.
